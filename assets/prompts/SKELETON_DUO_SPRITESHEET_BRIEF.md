# Arena Duel - Skeleton Duo Spritesheet Brief

This brief is designed for image generation or external sprite production.

## Goal

Create 2 high-quality 2D game skeleton characters as clean, readable fantasy arena sprites for Arena Duel.

The output should be suitable for a top-down or pseudo-top-down combat game with strong gameplay readability.

## Shared production constraints

- transparent background
- very clean silhouette
- readable anatomy from gameplay distance
- strong shape language
- polished and aligned animation frames
- no blur
- no realism
- no gore
- no photoreal rendering
- fantasy arcade feel
- old-school action RPG readability
- Ragnarok-like clarity without copying any existing IP
- consistent proportions across all directions
- game-ready sprite presentation

## Sprite sheet requirements

- 2 different skeleton characters
- each character must include:
- front view
- back view
- left view
- right view
- multiple walking animation frames for every direction
- aligned feet / ground contact across frames
- consistent frame spacing
- each direction easy to identify
- each character clearly separated from the other

## Recommended layout

- 1 organized master sprite sheet
- top half: skeleton fighter A
- bottom half: skeleton fighter B
- for each character:
- 4 rows = front, back, left, right
- 4 columns = walk_0, walk_1, walk_2, walk_3
- recommended frame size: 64x64
- recommended internal padding per frame: 6 to 8 px
- anchor: center_bottom

## Character A - Bastion Ember

Role:
- agile skeleton duelist
- fast arena fighter
- ember-gold accent identity

Visual traits:
- slim but readable torso
- slightly larger shoulder silhouette
- short fantasy shoulder guard on one side
- ember sash or cloth accent
- small curved sword or short arena blade silhouette
- warm gold-orange eye glow
- strong triangular silhouette

Prompt:

```text
Use case: stylized-concept
Asset type: game character sprite sheet
Primary request: create a polished 2D fantasy arena skeleton fighter sprite sheet for a game-ready action RPG character
Scene/backdrop: transparent background only
Subject: skeleton warrior A, agile arena duelist, slim bones, compact rib cage, short asymmetrical shoulder armor, ember sash, short blade silhouette, glowing orange-gold eyes
Style/medium: high-quality 2D sprite art, crisp pixel-art or ultra-clean hand-painted sprite style, not blurry, gameplay readable
Composition/framing: organized sprite sheet, 4-direction views with aligned walking frames, front back left right, 4 walk frames for each direction, consistent proportions and fixed foot alignment
Lighting/mood: bright fantasy arcade readability, clear contrast, combat-ready
Color palette: bone ivory, deep navy shadows, ember gold, warm orange accents
Constraints: transparent background, clean silhouette, readable anatomy, no blur, no gore, no realism, no watermark, no text, no background scene
Avoid: muddy rendering, painterly blur, realistic bones, horror tone, extra props, inconsistent scale, uneven frame alignment
```

Use this if a single prompt must generate the full organized sheet:

```text
Use case: stylized-concept

Primary request: generate a clean organized sprite sheet containing 2 different skeleton fighter characters for a fantasy arena combat game
Scene/backdrop: transparent background only
Subject: two distinct skeleton warriors, character A agile ember duelist, character B sturdy cyan guardian
Style/medium: high-quality 2D sprite art with old-school action RPG readability, crisp pixel-art or very clean sprite style, not blurry

```

## Acceptance checklist

- both characters are clearly different at first glance
- all 4 directions are easy to identify
- left and right are mirrored only if anatomy and gear remain readable
- feet remain aligned across the walk cycle
- silhouettes read well at small size
- no black background
- no glow haze around the characters
- no frame-to-frame scale drift
- no detached props or noisy accessories
